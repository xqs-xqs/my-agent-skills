# 🏎️ 算法速刷引擎 (Algorithm Speed-Run Engine)

> 收到题目，立刻一次性输出：物理模型 → 核心铁律 → 朴素+极客双代码 → 交互可视化 → 灵魂拷问。
> 无阻断，无等待，直接拿走完整高质量题解。

---

## 和深度版 (`algo-deconstruct-engine`) 的区别

| | 速刷版 (algo-speed-run) | 深度版 (algo-deconstruct-engine) |
|--|--|--|
| **输出节奏** | 单轮，一次给完 | 两轮，先盲推再解析 |
| **盲推阻断** | ❌ 无 | ✅ 硬阻断，强制先作答 |
| **冷启动测试卷** | ❌ 无 | ✅ 生成一周后可用的测试卷 |
| **可视化** | ✅ 有 | ✅ 有 |
| **适合场景** | 速刷、突击、查漏补缺 | 深度学习、长期留存 |
| **单题耗时** | 快（一次对话） | 慢（两轮对话+盲推思考） |

**用哪个？** 面试前三天突击用速刷版。平时想真正掌握一道题用深度版。

## 五模块一览

### 模块一：物理模型构建
完全抛开编程语言，用现实世界的物理现象映射数据结构。
解释暴力解的痛点 + 点透「Aha! Moment」。

示例比喻：
- 单调栈 → 俄罗斯套娃 / 叠盘子，遇矮结算
- 双指针 → 楚河汉界，相向推进
- 滑动窗口 → 传送带上的取景框
- BFS → 水面涟漪扩散

### 模块二：核心法则
提炼 1-3 条「可迁移铁律」，有记忆点、有动作感。
指出最反直觉的一个致命边界条件。

示例铁律：
- 单调栈：「遇到矮子就疯狂结算」
- 滑动窗口：「窗口内脏了就收缩」
- 二分答案：「猜一个答案，问它是否可行」

### 模块三：双层代码
先给**朴素版**（标出重复计算的元凶），再给**极客版**（物理命名 + 致命注释）。
精准分析复杂度，打破常见误区。

### 模块四：交互式可视化
可点击的「下一步 / 上一步 / 重置」动态推演组件。
初始用例特意选能触发边界的输入（如 `[2,2,2,1]`），而不是一眼就能看出答案的平庸数据。

### 模块五：灵魂拷问
三连追问，每类各一个：
1. **极端用例打击**：全相同/倒序/空数组会退化吗？
2. **限制条件改变**：禁止额外空间/单次遍历/数据流，思路怎么变？
3. **工程化映射**：这个算法在数据库/中间件/操作系统里的真实应用

## 设计说明

**为什么保留朴素版代码？**
因为「极致优雅」是消费者视角。
一步甩给你完美代码，你会以为好代码是天才一蹴而就的。
看到朴素版 → 极客版的跨越，你才能理解「降维」是什么感觉。

**为什么保留可视化？**
可视化是速刷版唯一和「看别人的题解文章」拉开差距的地方。
文字描述算法过程永远比不上「自己点下一步看动画跑一遍」。
哪怕是速刷，这一步的时间投入也是值得的。

**为什么没有冷启动测试卷？**
速刷的定位决定了：你现在需要的是「快速拿到高质量题解」。
「一周后还记不记得」是深度版解决的问题。
两个 Skill 加在一起才是完整的刷题体系。
