# 🚀 算法解构引擎 (Algorithm Deconstruction Engine)

> 一个反"看懂即会"的算法题深度刷题 Skill。
> 通过**物理直觉 + 盲推阻断 + 冷启动测试**，让算法真正"长"在你脑子里。

---

## 为什么做这个 Skill

刷题最致命的陷阱，不是题目太难，而是**"用看懂代替长出来"**。

你打开一份精美题解，边读边点头："卧槽这个单调栈的比喻绝了！"
三天后换一道题，你发现自己依然写不出来。

因为你记住的是**别人的比喻**，不是**你自己推演出来的路径**。

这个 Skill 的设计目标，就是强制打断这个"看完就爽，但什么都没留下"的循环。

---

## 核心特性

### 🛑 硬阻断盲推（最重要）

收到题目后，Skill **不会**立刻给你讲解。
它会先抛出 3 个面试官级别的灵魂问题，让你**独立作答**：

1. **数据规模嗅探**：数据范围暗示了什么复杂度？
2. **暴力解自白**：你的第一反应暴力解是什么？
3. **浪费点定位**：暴力解的重复计算发生在哪里？

**答完才给解析**。这一步是本 Skill 的灵魂，请不要跳过。

### 🧊 物理直觉建模

全面拒绝术语堆砌。
所有算法都用现实世界的物理现象映射：
- 单调栈 → 俄罗斯套娃 / 叠盘子游戏
- 双指针 → 楚河汉界相向推进
- 滑动窗口 → 传送带上的取景框
- BFS → 水面涟漪扩散
- 并查集 → 朋友圈合并

### 💎 朴素版 + 极客版双代码

不一步到位甩给你"极客代码"。
先给你一版朴素的 O(n²)，再给你优化后的 O(n)。
**对比才有降维的视觉冲击**，也让你明白"优化"不是天才一蹴而就的魔法。

### 🎬 交互式可视化

生成可点击的"下一步 / 重置 / 回退"可视化组件，
用动画演示算法的动态执行过程。
测试用例特意选取**能触发所有边界条件**的输入（如 `[2,2,2,1]`），
而不是教科书式的平庸输入。

### 🧠 灵魂拷问三连

大厂面试官最爱的三类追问：
1. **极端用例打击**：全相同 / 全倒序 / 空数组会不会退化？
2. **限制条件改变**：禁止额外空间 / 数据流场景思路怎么变？
3. **工程化映射**：这个思想在数据库索引、中间件源码里哪用到了？

### 📅 一周后冷启动测试卷

本 Skill 最大的"反遗忘曲线"武器。

每次学完一道题，Skill 会生成一份**独立成段的测试卷**：
- 📝 空白作答区（物理模型 / 伪代码 / Aha Moment / 边界）
- ✅ 折叠的参考答卷（防偷看）
- 📊 自我评分 Rubric（10 分制）
- 🚦 档位对应行动（9-10 分结业 / 6-8 分重做 / 3-5 分重学）

**一周后**把测试卷单独复制出来用，独立复盘，不需要再找 Claude。

## 使用方式

### 触发方式

在 Claude 中粘贴任何算法题，或说出以下任一表述：

- "帮我解构一下这道题：[题目]"
- "这道题怎么做：[LeetCode 链接或描述]"
- "分析一下这道题"
- 直接粘贴题目内容

### 完整流程示例

**第一轮对话**（你贴题 → Claude 只问三连问）

```
你：
  力扣 42：接雨水
  [题目描述...]

Claude：
  （以面试官身份开场）
  在我展开解析之前，先请你回答三个问题：
  
  1. 【数据规模嗅探】n ≤ 2×10^4，你认为目标复杂度是...
  2. 【暴力解自白】你的第一反应暴力解是什么？
  3. 【浪费点定位】暴力解的重复计算发生在哪？
  
  请先作答，答完我再展开完整解构。
```

**第二轮对话**（你作答 → Claude 展开完整解析）

```
你：
  1. 应该是 O(n) 或 O(n log n)
  2. 每个格子单独算左右最大值，O(n²)
  3. 重复扫描了左右最大值

Claude：
  【盲推点评】前两问抓得准，第三问只说了半截——
  真正的浪费是：每个位置都要重新扫一遍整个数组找两侧最大值。
  这就是我们即将"降维"的点。
  
  【模块一：物理模型构建】
  想象一个横截面...[楚河汉界 / 水位线比喻]
  
  【模块二：核心法则】
  ...
  
  【模块三：朴素版 + 极客版双代码】
  ...
  
  【模块四：交互式推演器】
  [生成可视化组件]
  
  【模块五：灵魂拷问】
  ...
  
  【模块六：冷启动测试卷】
  [生成一份 7 天后可用的独立测试卷]
```

## 设计哲学（可选阅读）

### 反模式识别

本 Skill 被设计用来对抗以下常见刷题反模式：

| 反模式 | 本 Skill 的对策 |
|--------|----------------|
| 看题解 10 秒就"懂了" | 硬阻断：不作答不给解析 |
| 只记代码不记思路 | 物理直觉建模 + 禁用术语解释 |
| 第一次一步到位看完美代码 | 朴素版 + 极客版对比呈现 |
| 学完就忘 | 一周后冷启动测试卷 |
| 只会原题不会变题 | 灵魂拷问三连（极端/限制/工程化） |

### 三个时间尺度的学习

本 Skill 配合以下学习节奏使用效果最佳：

- **单题尺度（当天 60-90 分钟）**：盲做 → 丑代码 → 用 Skill 解构 → 闭眼口述
- **周尺度（每周 1 小时）**：按算法家族重新串联本周题目
- **月尺度（每月 1 小时）**：画题型地图，标注变异点

### 为什么不用"极致优雅的代码"作为主视角

因为"极致优雅"是**消费者视角**。
面试现场你写出来的一定是丑代码，能力在于**在丑代码上迭代**。
所以本 Skill 刻意保留"朴素版"，让你看清迭代的路径，而不是误以为好代码是天才一次写出来的。

## 常见问题

**Q：为什么非要两轮输出？一次给完不是更方便吗？**
A：方便是学习的敌人。研究表明"先测试再学习"（testing effect）的留存率是纯阅读的 3 倍以上。如果你实在赶时间，那就是在用长期效果换短期效率。

**Q：盲推我真的什么都想不出怎么办？**
A：写"不知道"也算回答。有思考痕迹就够。你**意识到自己卡在哪里**，比你猜对答案更有价值。

**Q：冷启动测试卷一周后我真的会用吗？**
A：如果你不用，那说明你其实没打算"真正掌握"这道题，只是想要"学完的满足感"。诚实面对这个问题，比假装勤奋更重要。

**Q：可以用来学习非算法题吗（如系统设计）？**
A：不推荐。本 Skill 的结构高度针对"有标准解、有复杂度量化、有代码可验证"的算法题。系统设计需要另一套 Skill。

>
> 这个能力一旦具备，它会迁移到你工作中遇到的每一个性能瓶颈、每一次架构决策、每一场技术辩论。
>
> 这才是值得花时间的事。
