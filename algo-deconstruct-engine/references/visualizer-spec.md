# 模块四：交互式推演器构建规范

## 工具选择

优先使用 `visualize:show_widget` 工具生成 HTML 交互式组件。
在调用前先调用 `visualize:read_me` 加载 `interactive` 模块获取最新规范。

## 必须满足的五要素

每个推演器都必须清晰包含以下五项：

### 1. Objective（目标）
一句话说明演示的核心机制。

**好的示例**：
- "可视化单调栈在柱状图问题中的'遇矮结算'机制"
- "演示双指针在有序数组中如何通过相向碰撞将 O(n²) 降为 O(n)"

**差的示例**：
- "可视化这个算法" → 太泛
- "展示代码执行过程" → 重点错了，重点是机制不是代码

### 2. Data State（数据状态）
必须选择一个**能触发边界条件**的测试用例，而不是"一眼就能看出答案"的平庸输入。

**选取原则**：
- 包含连续重复元素（如 `[2,2,2,1]`）
- 包含极端值（首尾最大、首尾最小）
- 长度不要超过 8，否则动画太长用户失去耐心
- 长度不要小于 4，否则覆盖不到有趣情况

### 3. Strategy（策略）
默认使用清晰的水平布局：
- 顶部：数据结构的可视化（柱状图、数组格子、链表节点等）
- 中部：指针/栈等辅助结构的状态
- 底部：当前步骤的解释文字 + 控制按钮

### 4. Inputs（控制输入）
必须提供以下控制：
- **下一步 (Next Step)**：主按钮
- **重置 (Reset)**：回到初始状态
- **上一步 (Prev Step)**：可选但强烈推荐，方便用户回看
- **自动播放 (Auto Play)**：可选，2 秒/步

### 5. Behavior（视觉行为描述）
用**动作语言**描述，不要写死具体颜色/字号/像素值。

**好的动作描述**：
- "当指针相遇时，高亮显示该区域，弹出计算面板展示公式 `面积 = 宽 × 高`，然后左指针向右平移一格"
- "栈顶元素被弹出时，该柱子整体变灰并向上飘出视野"
- "新元素入栈时，从数组位置滑动到栈顶位置"

**差的动作描述**：
- "把当前元素染成 #FF5733" → 写死颜色
- "显示结果" → 太笼统

## 代码结构建议

```html
<div id="container">
  <!-- 数据结构区 -->
  <div id="data-viz"></div>
  
  <!-- 辅助结构区（栈、指针、窗口等） -->
  <div id="aux-viz"></div>
  
  <!-- 说明文字 -->
  <div id="explanation"></div>
  
  <!-- 控制按钮 -->
  <div id="controls">
    <button onclick="prevStep()">上一步</button>
    <button onclick="nextStep()">下一步</button>
    <button onclick="reset()">重置</button>
  </div>
</div>

<script>
  // 核心：维护一个 steps 数组，每步记录完整状态
  const steps = [
    { data: [...], stack: [...], pointers: {...}, message: "..." },
    // ...
  ];
  let currentStep = 0;
  
  function render() { /* 根据 steps[currentStep] 重绘 */ }
  function nextStep() { if (currentStep < steps.length-1) { currentStep++; render(); } }
  // ...
</script>
```

## 交互反馈原则

- **每步都要有说明文字**：解释"为什么现在做这个动作"
- **关键时刻弹提示**：如"🎯 Aha Moment: 遇到矮子，开始结算！"
- **最终步要展示结果**：不要停留在中间状态
- **颜色语义化**（用 CSS 变量）：
  - 活跃元素 = 主色调
  - 已处理元素 = 淡化
  - 即将被淘汰 = 警示色
  - 结果/答案 = 成功色

## 常见算法的可视化模板参考

| 算法类型 | 可视化重点 |
|---------|-----------|
| 双指针 | 两个箭头在数组上移动，相遇/错开时有动画 |
| 滑动窗口 | 一个半透明矩形框在数组上滑动，框内聚合值实时更新 |
| 单调栈 | 右侧栈区 + 左侧数据区，入栈出栈有滑动动画 |
| 二分查找 | 左右边界收缩，mid 指针闪烁 |
| BFS | 以起点为中心的波纹扩散效果 |
| DFS | 深度优先的路径描边 + 回溯时擦除 |
| DP | 网格逐格填充，箭头指示依赖方向 |
| 回溯 | 决策树展开 + 失败分支变灰 |
